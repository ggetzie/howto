Instructions

  1. Install System dependencies

    sudo apt-get install python3-dev python3-setuptools libatlas-base-dev gfortran \ 
	  	libfreetype6-dev libpng-dev libjpeg-dev git nginx makepasswd postgresql postgresql-contrib \
	  	supervisor libxml2-dev libxslt1-dev python-psycopg2 libpq-dev

  2. Create app user

    Create the webapps group if it doesn't exist. 

    sudo groupadd --system webapps
    sudo useradd --system --gid webapps --shell /bin/bash --home /home/username username

  3. Set up the virtualenv

    Set up the virtualenv wherever environments are stored. This should be outside the repo. 

    cd /path/to/env/
/usr/bin/python3 -m virtualenv --python=/usr/bin/python3 aeqenv

    Add an alias to .bashrc to activate the environment. 

    alias aeqenv="source /path/to/env/aeqenv/bin/activate"

    Add environment variables to the virtualenv, use script setup-keys 

    setup-keys project-slug environment-directory

  4. Set up the git repository

    Clone an existing repository

    git clone ...

    Start a new one with cookiecutter

    cookiecutter https://github.com/pydanny/cookiecutter-django

  5. Install python packages via pip

    requirements.txt should be in the top level of the git repository. It should include numpy, scipy, scikit-learn, matplotlib, django, pandas, gunicorn, psycopg2 and their dependencies. 

     pip install -r requirements/requirements.txt

  6. Create database

    		sudo su postgres
	  	createuser --interactive -P
	  	Enter name of role to add: aeq_user
	  	Enter password for new role:
	  	Enter it again:
	  	Shall the new role be a superuser? (y/n) n
	  	Shall the new role be allowed to create databases? (y/n) y
	  	Shall the new role be allowed to create more new roles? (y/n) y

    alternatively: 
    psql postgres -c "CREATE ROLE db_user WITH LOGIN PASSWORD '${DB_PW}';"
psql postgres -c "CREATE DATABASE db_name OWNER db_user;"

  7. Set up settings files

    This part is only necessary for starting from scratch. Make new directory, separate base from files for local, staging, production, etc. This will be set up already if you started with django-cookiecutter. 

    cd /project/prophit_main/
mkdir settings
mv settings.py settings/base.py	 
cd settings
touch __init__.py

    Add nifty function for getting environment variables. (already included with cookiecutter). 

    from django.core.exceptions import ImproperlyConfigured

def get_env_variable(var_name):
    """Get the environment variable or return exception"""
    try:
        return os.environ[var_name]
    except KeyError:
        error_msg = "Set the {} environment variable".format(var_name)
        raise ImproperlyConfigured(error_msg)

    This has to be done for each deploy: Add a line to the virtualenv activate script to set the DJANGO_SETTINGS_MODULE environment variable for our new settings structure. 

    export DJANGO_SETTINGS_MODULE="config.settings.local"

  8. Configure Database

    Note, this is only necessary for setting up from scratch. Settings will be ready if set up from cloned repo. 

    DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql_psycopg2',
        'NAME': 'db_name',
        'USER': 'db_user',
        'PASSWORD': get_env_variable(DB_PW) # store in environment variable
        'HOST': 'localhost',
        'PORT': '',                      # Set to empty string for default.
    }
}

    Finally, apply the database migrations 

    python manage.py migrate

  9. Create directory for logs

    This should be in the project directory, but excluded from source control in .gitignore

    Run commands in the project directory

    mkdir logs
touch logs/debug.log
touch logs/gunicorn_supervisor.log
touch logs/nginx-access.log
touch logs/nginx-error.log

  10. Configure gunicorn

    The script below should be included in a cloned repo as gunicorn_start_local.sh. Ensure filepaths are still correct on the new deploy

     #!/bin/bash
NAME="app_name"
DJANGODIR="/usr/local/src/project-slug/"
SOCKFILE="/usr/local/src/project-slug/run/gunicorn.sock"
USER=app_user
GROUP=webapps
NUM_WORKERS=3
DJANGO_SETTINGS_MODULE=config.settings.local
DJANGO_WSGI_MODULE=config.wsgi

echo "Starting $NAME as `whoami`"

# Activate the virtual environment
cd $DJANGODIR
source /usr/local/src/env/envdir/bin/activate

export DJANGO_SETTINGS_MODULE=$DJANGO_SETTINGS_MODULE
export PYTHONPATH=$DJANGODIR:$PYTHONPATH

# Create the run directory if it doesn't exist
RUNDIR=$(dirname $SOCKFILE)
test -d $RUNDIR || mkdir -p $RUNDIR

# Start you Django Unicorn 
# Programs meant to be run under supervisor should not
# daemonize themselves.
# (do not use --daemon)

exec /usr/local/src/env/envdir/bin/gunicorn ${DJANGO_WSGI_MODULE}:application \
    -- name $NAME \
    -- workers $NUM_WORKERS \
    -- user=$USER --group=$GROUP \
    --bind=unix:$SOCKFILE \
    --log-level=debug \
    --log-file=-

  11. Configure supervisor

    Supervisor monitors the gunicorn process and restarts it automtically when the machine reboots 

    [program:project-slug]
command = /usr/local/src/project-slug/bin/gunicorn_start_local.bash ; Command to start app
user = app_user                                                     ; User to run as
; Where to write log messages
stdout_logfile = /usr/local/src/project-slug/log/gunicorn_supervisor.log   
; Save stderr in the same log
redirect_stderr = true                                                
environment=LANG=en_US.UTF-8,LC_ALL=en_US.UTF-8                       ; Set UTF-8 as default encoding

    Keep the conf file in source control and symlink it to the supervisor conf directory

    cd /etc/supervisor/conf.d
sudo ln -s /usr/local/src/project-slug/srv/local/super_project-slug.conf
sudo supervisorctl reread
sudo supervisorctl update

    Add an alias to ~/.bashrc

    echo 'alias reproject-slug="sudo supervisorctl restart project-slug"' >> ~/.bashrc

  12. Configure nginx

    The settings below should be included in the repo as bin/nginx_local_aeq.conf

    upstream aeq_app_server {
  # fail_timeout=0 means we always retry an upstream even if it failed
  # to return a good HTTP response (in case the Unicorn master nukes a
  # single worker for timing out).
 
  server 127.0.0.1:8000 fail_timeout=0;
}
 
server {
 
    listen   80;
    server_name project-slug;
 
    client_max_body_size 4G;
 
    access_log /usr/local/src/project-slug/log/nginx-access.log;
    error_log /usr/local/src/project-slug/log/nginx-error.log;
 
    location /static/ {
        alias   /usr/local/src/project-slug/staticfiles/;
    }
    
    # location /media/ {
    #     alias   /usr/local/src/cashcal/media/;
    # }
 
    location / {
        # an HTTP header important enough to have its own Wikipedia entry:
        #   http://en.wikipedia.org/wiki/X-Forwarded-For
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
 
        # enable this if and only if you use HTTPS, this helps Rack
        # set the proper protocol for doing redirects:
        # proxy_set_header X-Forwarded-Proto https;
 
        # pass the Host: header from the client right along so redirects
        # can be set properly within the Rack application
        proxy_set_header Host $http_host;
 
        # we don't want nginx trying to do something clever with
        # redirects, we set the Host: header above already.
        proxy_redirect off;
 
        # set "proxy_buffering off" *only* for Rainbows! when doing
        # Comet/long-poll stuff.  It's also safe to set if you're
        # using only serving fast clients with Unicorn + nginx.
        # Otherwise you _want_ nginx to buffer responses to slow
        # clients, really.
        # proxy_buffering off;
 
        # Try to serve static files from nginx, no point in making an
        # *application* server like Unicorn/Rainbows! serve static files.
        if (!-f $request_filename) {
            proxy_pass http://prophit_app_server;
            break;
        }
    }
	  }

    Create a symlink to the config file under /etc/nginx/sites-enabled and restart nginx

    sudo ln -s /usr/local/src/aeq/bin/nginx_local_aeq.conf \
/etc/nginx/sites-enabled/aeq
sudo service nginx restart

    edit /etc/hosts to include a mapping from the project name to the local dev site

    sudo echo "127.0.0.1	project-slug" >> /etc/hosts
